Contributing to Kudu

There are lots of ways to get involved with the Kudu project. Some of them are listed below. You don't have to be a developer; there are lots of valuable and important ways to get involved that suit any skill set and level.

If you want to do something not listed here, or you see a gap that needs to be filled, let us know.

Participate in the community.

Community is the core of any open source project, and Kudu is no exception. Participate in mailing lists, requests for comment, chat sessions, bug reports, and other communication mechanisms. See Community to get started.

Talk about how you use Kudu.

Let us know what you think of Kudu and how you are using it. Send blogs or presentation links to kudu-user@cloudera.org so that we can feature them.

Write code.

You can write and submit code for the core Kudu project or extend your existing code base or API to work with Kudu. Follow the code guidelines to be sure your submission is easy to review and integrate.

File bugs and enhancement requests.

If you see problems in Kudu or you think a feature would make Kudu more useful to you, let us know by filing a bug or request for enhancement on the Kudu issue tracker. The more information you can provide about how to reproduce an issue or how you'd like a new feature to work, the better.

Review patches and test code.

In order for patches to be integrated into Kudu as quickly as possible, they must be reviewed and tested. The more eyes, the better. Even if you are not a committer, your input is extremely valuable. Keep an eye on the Kudu gerrit instance for patches that need review or testing.

Write and review documentation.

Good documentation makes great software even better. Where you see gaps in the documentation, submit suggestions, corrections, or documentation patches. You can also correct or edit error messages, log messages, or API documentation. If you'd like to translate the Kudu documentation into a different language or you'd like to help in some other way, let us know. Review the documentation guidelines before you get started.

Request and review examples.

The kudu-examples repository includes working code examples. As more examples are requested and added, they will need review and clean-up. This is another way you can get involved.
